# development-planning 0.1.0

## Summary

Initial sealed release of the `development-planning` harness workflow for turning PRD/HLD/LLD sources under `docs/` into a validated, planning-only development task DAG under `tasks/development`.

## Included

- Portable harness core files: `agent.md`, `workflow.md`, `rules.md`, and `manifest.yaml`.
- Runtime Codex adapter under `adapters/codex/`.
- Runtime skill entry and references under `skill/`.
- Development planning schema and package validator.
- Positive, negative, and boundary eval cases.
- Ready, blocked, and invalid validator fixtures.

## Validation

- `validators/validate_harness.py`
- `validators/validate_development_planning_package.py tests/fixtures/ready-package`
- `validators/validate_development_planning_package.py tests/fixtures/blocked-package`
- `tools/validate_skill_static.py`
- `tools/build_release.py --check`

Built artifacts are recorded in `dist/release-manifest.yaml`.
