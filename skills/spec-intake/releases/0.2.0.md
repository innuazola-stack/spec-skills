# Spec Intake 0.2.0

## Summary

This release hardens `spec-intake` as a harness-first, review-gated self-improving generation workflow.

It includes hard Stage 1 gates for `interaction_decision` and requirement-table provenance so clarification, blocked drafts, and bounded inference are auditable instead of prompt-only.

## Validation

- Source harness validation
- Source regression probes
- Output package regression probes
- Fixture, snapshot, and live-run package validation

## Release Artifacts

Release artifacts are generated by `tools/build_release.py` under `dist/`.
