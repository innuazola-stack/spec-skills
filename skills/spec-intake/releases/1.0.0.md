# Spec Intake 1.0.0

## Summary

This release seals `spec-intake` as a production-ready harness workflow for turning a raw idea into a structured requirement table, review-gated Human/Agent PRDs, and an implementation-ready HLD.

The workflow boundary is PRD plus HLD only. It does not produce task plans, task graphs, dependency edges, parallel groups, implementation task cards, or task decomposition artifacts.

## Quality Bar

- Stage 1 produces a validator-clean `contract-envelope.json` with explicit `interaction_decision`, canonical requirement table rows, closed-form clarification questions, source-backed facts, bounded inferences, assumptions, blockers, and execution topology when the idea is executor-like.
- Stage 2 renders an English execution-ready Agent PRD and a professional Simplified Chinese Human PRD brief from the requirement table, then blocks Stage 3 until explicit Human PRD approval is recorded.
- Stage 3 produces both `high-level-design.json` and `high-level-design.md`, backed by fixed HLD design gates and an independent `hld-semantic-review.json`.
- Ready HLD requires concrete control flow, data flow, data objects, interface contracts, state model, technical decisions, implementation design, and no-mock real-environment acceptance.
