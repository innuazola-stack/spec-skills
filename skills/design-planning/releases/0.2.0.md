# design-planning 0.2.0

## Summary

Promotes `design-planning` from a compatibility skill entrypoint to a packaged review-gated harness workflow.

## Changes

- Sets the default planning output root to `tasks/design`.
- Keeps detailed design documents fixed under target `docs/design`.
- Adds portable harness core files, eval cases, runtime adapters, harness validation, release validation, and versioned runtime packages.
- Hardens generated task fixtures as documentation-only detailed design tasks.
- Adds final `design_acceptance` task requirements and validator coverage.

## Validation

- `validators/validate_harness.py`
- `tools/validate_skill_static.py`
- `tools/build_release.py --check`
- `validators/validate_design_planning.py --project-root <target> --planning-json <target>/tasks/design/design-planning.json`

## Compatibility

The `skill/SKILL.md` entrypoint remains for Codex discovery. The harness core is the source of truth.
